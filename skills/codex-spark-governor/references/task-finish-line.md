# Task finish line

Use this file for long tasks in any repo.

Read the repo's own instructions and standards first when they exist.

Before broad implementation starts:
- define the exact requested scope
- define what proof will count as done
- define any repo-specific checks that must pass

Do not stop at the first working draft.

Finish all of these when they apply:
- the requested behavior
- tests or real proof
- docs or usage notes for changed behavior
- examples or outputs that help users verify the result
- cleanup of stale copied files or drift
- one fresh final review

If the repo has its own coverage file, migration checklist, release checklist, or approval flow, those win.

If the task promises a full result, do not say `done` until the proof for that claim matches the work.
